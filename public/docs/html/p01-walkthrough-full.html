# Project Plan — P01 Task Tracking System

## Overview
A simple Kanban-style task management app for small teams.
Users can create projects, assign tasks, and track progress visually.
Supports English and Thai. Built to be genuinely enterprise-ready by the end.

## Timeline: Flexible, no fixed deadline

---

## Phase 1 — Project Setup + Auth
**Goal**: App runs locally, users can register and login, supports EN/TH

- [ ] Install Node.js, VS Code, Git
- [ ] Create Next.js project (`npx create-next-app@latest`)
- [ ] Copy `.ai/` folder into project
- [ ] Fill in AGENTS.md and CLAUDE.md (project rules, i18n rule, code comment rule)
- [ ] Setup internationalization (next-intl) — EN/TH — **before building more pages**
- [ ] Setup PostgreSQL + Prisma
- [ ] Create User model in database
- [ ] Add NextAuth.js (Google login or email/password)
- [ ] Build Login and Register pages (with EN/TH translations)
- [ ] Push first commit to GitHub

**Done when**: You can open the app in either language and log in

---

## Phase 2 — Projects & Tasks (CRUD)
**Goal**: Users can create projects and add tasks

- [ ] Create Project model (id, name, description, userId)
- [ ] Create Task model (id, title, status, projectId, assignedTo)
- [ ] Build API routes: GET/POST/PUT/DELETE for tasks
- [ ] Build project list page
- [ ] Build task creation form

**Done when**: You can create a project and add tasks to it, in both languages

---

## Phase 3 — Kanban Board
**Goal**: Drag and drop tasks between columns

- [ ] Install `@dnd-kit/core` library
- [ ] Build Kanban board UI (Todo / In Progress / Done columns)
- [ ] Add drag-and-drop to move tasks between columns
- [ ] Save new status to database when task is moved

**Done when**: You can drag a task from Todo to Done

---

## Phase 4 — Team Features
**Goal**: Multiple users can work on the same project

- [ ] Invite members to a project
- [ ] Assign tasks to specific team members
- [ ] Show who is assigned to each task
- [ ] Add email notification when task is assigned (Resend API)

**Done when**: Two accounts can collaborate on one project

---

## Phase 5 — Polish & Details
**Goal**: App feels complete and professional

- [ ] Add task due dates and priority levels
- [ ] Add task comments / notes
- [ ] Build project dashboard (task count, completion %)
- [ ] Mobile responsive design
- [ ] Loading states and error messages
- [ ] Security review with Claude Code before deploying

**Done when**: App looks and feels ready for real users

---

## Phase 6 — Deploy to Production
**Goal**: App is live on the internet (MVP-live)

- [ ] Setup Railway account and deploy PostgreSQL
- [ ] Deploy backend to Railway
- [ ] Deploy frontend to Vercel
- [ ] Set all environment variables
- [ ] Run `prisma migrate deploy`
- [ ] Test full flow on live URL, both languages
- [ ] Share link with at least 1 real person

**Done when**: Someone else can use your app from their phone — MVP is live,
but not yet enterprise-ready. Move to Phase 7 once you have real user feedback.

---

## Phase 7 — Enterprise Hardening
**Goal**: App meets what a real organization expects before paying for it

- [ ] Rate limiting + optional 2FA
- [ ] Audit log system
- [ ] Error monitoring (Sentry)
- [ ] Database backups + uptime monitoring
- [ ] Pagination, search, and database indexing
- [ ] PDPA compliance (consent, data export, account deletion)
- [ ] Admin panel (separate from regular users)
- [ ] Billing / subscription (Stripe or Omise) — only if selling as SaaS
- [ ] Onboarding flow + support channel
- [ ] Final enterprise readiness review with Claude Code

**Done when**: You'd feel comfortable selling this to a real company.

```

- id, name, email, password, role (USER/ADMIN), createdAt

Project
- id, name, description, ownerId (→ User), createdAt

ProjectMember
```

## Coding Rules
- Use **TypeScript** for all files
- Use **async/await** (not .then().catch())
- Always validate input before saving to database
- Never trust data from the frontend — verify on the server
- Use Prisma for ALL database queries (no raw SQL)
- Commit to GitHub after every completed feature
- **Never hardcode user-facing text** — always use next-intl translation keys
- **Comment every new or changed piece of code** — see AGENTS.md "Code Comments"

---

## Naming Conventions
- Files: `kebab-case.ts` (e.g. `task-card.tsx`)
- Components: `PascalCase` (e.g. `TaskCard`)
- Functions: `camelCase` (e.g. `createTask`)
- Database tables: `PascalCase` in schema (e.g. `Task`)
- API routes: `/api/projects`, `/api/tasks`
- Translation keys: `namespace.key` (e.g. `tasks.createButton`)
