ทางเลือกแทน IWS — ต่อ pub400 ด้วย ODBC เอง

สำหรับกรณีที่ไม่มีสิทธิ์ deploy IWS (ทั้งที่บริษัทและที่ pub400) แต่อยากพัฒนาแยกต่างหากด้วยตัวเอง

ทำไม IWS ใช้ไม่ได้

IWS ต้องมีคนตั้งค่า Web Services Server Instance ผ่าน IBM Web Administration for i ซึ่งต้องใช้สิทธิ์ authority ระดับสูง (ใกล้ admin) — บัญชี pub400 ทั่วไปที่สมัครฟรีมักได้แค่:

สิทธิ์ที่ได้ใช้ทำอะไรได้
5250 terminal (green screen)เขียน RPG, รัน CL command, จัดการไฟล์ตัวเอง
Database access (ODBC/JDBC)ต่อฐานข้อมูล DB2 for i จากเครื่องภายนอกได้โดยตรง
❌ HTTP Admin / IWS Studioไม่ได้ให้ทั่วไป เพราะกระทบ server instance ทั้งเครื่องที่ใช้ร่วมกับคนอื่น

ทางเลือก: ต่อฐานข้อมูลตรง แล้วสร้าง API เอง

แทนที่จะให้ IBM i เป็นคนสร้าง REST API ให้ (แบบ IWS) เราต่อเข้า database ตรงๆ ด้วย ODBC จากเครื่องเราเอง แล้วเขียน Node.js API เอง — ควบคุมได้ 100% ไม่ต้องพึ่งสิทธิ์ admin ของใครเลย

เทียบกับของบริษัท: Node.js API ที่คุณเขียนเองตรงนี้ ทำหน้าที่แทน "IWS" — เป็นตัวกลางแปลง SQL query ให้กลายเป็น REST endpoint เหมือนกัน แค่คุณเป็นคนคุมเองแทน

1
สมัคร pub400
  • เข้า pub400.com สมัครบัญชีฟรี
  • จะได้ user profile + สิทธิ์ database พื้นฐาน
  • จดค่าที่ต้องใช้ไว้: hostname, username, password
⚠️ ใช้ข้อมูลสมมติในการฝึกเท่านั้น ห้ามเอาข้อมูลบริษัทมาใส่ในตารางที่ pub400 เด็ดขาด
2
ติดตั้ง ODBC Driver บน PC

โหลด IBM i Access Client Solutions (ACS) จากเว็บ IBM (ฟรี) แล้วติดตั้งส่วน ODBC Driver — ใช้ได้ทั้ง Windows/Mac

ตั้งค่า DSN (Data Source Name) ชี้ไปที่ hostname ของ pub400 พร้อม username/password ที่สมัครไว้

3
ทดสอบเชื่อมต่อก่อนเขียนโค้ด

ใช้เครื่องมือทดสอบ ODBC ที่มากับ ACS ลอง query ง่ายๆ ก่อน เช่น

SELECT * FROM QSYS2.SYSTABLES FETCH FIRST 5 ROWS ONLY

ถ้าได้ผลลัพธ์กลับมา แปลว่าเชื่อมต่อสำเร็จ พร้อมไปขั้นต่อไป

4
สร้าง Node.js API ของตัวเอง
mkdir my-iws-alternative
cd my-iws-alternative
npm init -y
npm install express odbc cors dotenv
// .env
DB_DSN=PUB400
DB_USER=your_username
DB_PASSWORD=your_password
// server.js
require('dotenv').config();
const express = require('express');
const odbc = require('odbc');
const cors = require('cors');

const app = express();
app.use(cors());   // เปิด CORS ให้ Next.js เรียกได้ (ทำเองได้เต็มที่ ต่างจาก IWS ที่ต้องขอแอดมิน)

const connectionString =
  `DSN=${process.env.DB_DSN};UID=${process.env.DB_USER};PWD=${process.env.DB_PASSWORD}`;

app.get('/api/employee', async (req, res) => {
  const empid = req.query.empid;
  try {
    const connection = await odbc.connect(connectionString);
    const result = await connection.query(
      `SELECT * FROM MYLIB.EMPLOYEE WHERE EMPID = '${empid}'`
    );
    await connection.close();
    res.json(result);
  } catch (err) {
    res.status(500).json({ error: err.message });
  }
});

app.listen(3001, () => console.log('API running on http://localhost:3001'));
🔴 ระวัง SQL Injection: ตัวอย่างข้างบนต่อ string ตรงๆ (ไม่ปลอดภัย) ใช้ได้แค่ตอนฝึกส่วนตัว ถ้าจะทำจริงต้องใช้ parameterized query แทน เช่น connection.query(sql, [empid])
5
ต่อกับ Next.js (pattern เดียวกับที่เคยทำ)
// lib/config.ts
export const API_CONFIG = {
  BASE_URL: "http://localhost:3001/api",   // ชี้ไปที่ Node.js API ที่สร้างเอง แทน IWS
  ENDPOINTS: {
    EMPLOYEE: "/employee"
  }
};

ที่เหลือ (fetch(), hooks, component) ใช้ pattern เดิมทุกอย่างที่เคยทำกับบริษัทได้เลย ไม่ต้องแก้อะไรเพิ่ม — เพราะจากมุมมอง frontend มันคือ REST API เหมือนกัน แค่คนละคนสร้าง

เทียบ IWS กับวิธีนี้

IWS (บริษัท)Node.js + ODBC (วิธีนี้)
ใครสร้างแอดมิน/ทีม RPGคุณเอง
ต้องมีสิทธิ์ admin ไหมต้องมีไม่ต้อง (แค่ database access พื้นฐาน)
รันอยู่ที่ไหนบน IBM iบนเครื่อง PC ของคุณ (ต้องเปิดค้างไว้ตอนใช้งาน)
ควบคุม CORS/security เองไม่ได้ ต้องขอแอดมินได้เต็มที่
ใช้ฝึก pattern fetch/hooks/config✅✅ เหมือนกัน
สรุป: วิธีนี้ฝึก "หน้าบ้าน" (frontend, fetch pattern, state management) ได้เหมือนกันทุกอย่าง สิ่งเดียวที่ต่างคือ "หลังบ้าน" ไม่ใช่ IWS จริง — พอกลับไปทำงานที่บริษัท แค่เปลี่ยน BASE_URL กลับไปเป็น IWS endpoint จริง โค้ด frontend แทบไม่ต้องแก้เลย

เรื่องความปลอดภัยที่ต้องรู้

  • ห้าม commit ไฟล์ .env (มี password) ขึ้น GitHub — ใส่ใน .gitignore เสมอ
  • Node.js API ที่สร้างเองรันบน PC เท่านั้น ไม่ได้ deploy public ให้คนอื่นเข้าถึง เหมาะกับฝึกส่วนตัวเท่านั้น
  • ถ้าจะทำ query จริง ใช้ parameterized query เสมอ ป้องกัน SQL Injection
  • Credential ของ pub400 เป็นของส่วนตัว ห้ามแชร์ให้คนอื่น
