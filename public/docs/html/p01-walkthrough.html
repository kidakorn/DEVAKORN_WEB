P01 Task Tracking
System

A complete step-by-step example from empty folder to a live app. Use this as a template for every project after this.

Next.js 14 PostgreSQL Prisma NextAuth Railway Vercel
00

Before You Start

Make sure these are installed and ready before touching any code.
1
Install Node.js
Download from nodejs.org — choose the LTS version (the one that says "Recommended for most users")
Terminal — verify install
$ node --version
v20.x.x ← should show v18 or higher
$ npm --version
10.x.x
2
Install Git + Create GitHub Account
git-scm.com to download Git. Then create a free account at github.com if you don't have one.
Terminal — verify + setup
$ git --version
git version 2.x.x
$ git config --global user.name "Your Name"
$ git config --global user.email "your@email.com"
3
Login Claude Code in VS Code
Open VS Code → Click the Claude Code icon on the left sidebar → Click "Claude.ai Subscription" → Login with your account.
✓Ready when: You can see the Claude Code chat panel open inside VS Code.
4
Create GitHub Repository
Go to github.com → Click "New" → Name it task-tracking-system → Set to Public → Do NOT add README → Click "Create repository" → Copy the repo URL.
PHASE 1

Project Setup + Auth

By the end of Phase 1 — app runs locally, users can register and login, code is on GitHub.
1
Create the Next.js Project
Open VS Code → Open Terminal (Ctrl + `) → Run these commands:
Terminal
$ npx create-next-app@latest task-tracking-system

✔ Would you like to use TypeScript? → Yes
✔ Would you like to use ESLint? → Yes
✔ Would you like to use Tailwind CSS? → Yes
✔ Would you like your code inside a src/ directory? → Yes
✔ Would you like to use App Router? → Yes
✔ Would you like to use Turbopack? → No
✔ Would you like to customize the import alias? → No

 $ cd task-tracking-system
$ code . ← opens project in VS Code
2
Copy .ai/ folder into the project
Copy your SKILLS.md, PLAN.md, CONTEXT.md into a folder called .ai/ inside the project. The structure should look like this:
Project Structure
task-tracking-system/
├── .ai/
│ ├── CONTEXT.md
│ ├── PLAN.md
│ └── SKILLS.md
├── src/
├── package.json
└── ...
3
Ask Claude Code to setup Database + Auth
Open Claude Code chat in VS Code (Ctrl + Alt + I) and paste this prompt:
📋 Paste this into Claude Code Chat
Read all files in .ai/ folder. I just created a new Next.js 14 project called task-tracking-system. I need to set up PostgreSQL + Prisma + NextAuth. Please give me step-by-step terminal commands to: 1. Install Prisma and set it up 2. Install NextAuth.js 3. Install shadcn/ui for components 4. Create the initial database schema for: - User (id, name, email, password, createdAt) - Project (id, name, description, ownerId, createdAt) - Task (id, title, status, priority, projectId, assignedTo, dueDate, createdAt) Give me every command to run in terminal, one at a time.
💡 Claude Code will give you exact commands. Run them one by one in terminal. If any command fails, paste the error back to Claude.
4
Setup .env file
Create a file called .env in the project root (not inside src/) with these values:
.env
DATABASE_URL="postgresql://postgres:password@localhost:5432/tasktracker"
NEXTAUTH_SECRET="any-random-string-here-make-it-long"
NEXTAUTH_URL="http://localhost:3000"
⚠️Important: Add .env to your .gitignore file so it never gets pushed to GitHub. It contains secrets.
5
Create the database schema
Ask Claude Code to write the Prisma schema for you:
📋 Paste into Claude Code Chat
Write the complete Prisma schema file for my Task Tracking app. Models needed: - User: id, name, email, password (hashed), createdAt - Project: id, name, description, owner (relation to User), members (many-to-many with User), createdAt - Task: id, title, description, status (TODO/IN_PROGRESS/DONE), priority (LOW/MEDIUM/HIGH), project (relation), assignedTo (relation to User), dueDate, createdAt Also write the command to run the first migration.
💡 Claude will write the full schema.prisma file and give you the migration command to run.
After Claude gives you the schema — run this
$ npx prisma migrate dev --name init
✔ Your database is now in sync with your schema.
6
Build the Login & Register pages
Ask Claude Code to build the auth pages:
📋 Paste into Claude Code Chat
Build the login and register pages for my Next.js app. Setup: - Using Next.js 14 App Router - NextAuth.js for authentication - Prisma for database - Tailwind CSS + shadcn/ui for styling I need: 1. /app/(auth)/login/page.tsx — login form (email + password) 2. /app/(auth)/register/page.tsx — register form (name + email + password) 3. /lib/auth.ts — NextAuth configuration 4. /api/auth/[...nextauth]/route.ts — NextAuth API route 5. /api/auth/register/route.ts — register new user API Keep the UI clean and simple. Show me which files to create and what to put in each one.
💡 Claude Code will create multiple files at once. Review each one before saving.
7
Test login works
Terminal
$ npm run dev
▲ Next.js 14.x.x
- Local: http://localhost:3000
What you should see
Open http://localhost:3000/register — fill in name, email, password → click Register
Open http://localhost:3000/login — use the same email + password → you're logged in
After login, you're redirected to the dashboard page
🐛Something broken? Paste the error message into Claude Code chat — it will tell you exactly what to fix.
8
Push to GitHub
Terminal
$ git init
$ git add .
$ git commit -m "feat: initial setup with auth"
$ git remote add origin https://github.com/YOUR_USERNAME/task-tracking-system.git
$ git push -u origin main
Phase 1 Complete — Users can register and login. Code is on GitHub.
PHASE 2

Projects & Tasks (CRUD)

By the end of Phase 2 — users can create projects and add tasks to them.
1
Build Project CRUD
📋 Paste into Claude Code Chat
Build the Projects feature for my Task Tracking app. I need: 1. /app/(dashboard)/projects/page.tsx — list all projects for the logged-in user 2. /app/(dashboard)/projects/new/page.tsx — form to create a new project 3. /app/(dashboard)/projects/[id]/page.tsx — view a single project with its tasks 4. /api/projects/route.ts — GET (list) and POST (create) endpoints 5. /api/projects/[id]/route.ts — GET, PUT (update), DELETE endpoints Rules: - Only show projects owned by the logged-in user - Protect all routes — redirect to login if not authenticated - Use Prisma for all database queries - Simple clean UI with Tailwind CSS
2
Build Task CRUD
📋 Paste into Claude Code Chat
Build the Tasks feature for my Task Tracking app. I need: 1. /api/tasks/route.ts — POST to create a new task 2. /api/tasks/[id]/route.ts — PUT (update status/details), DELETE 3. A TaskCard component — shows title, status badge, priority, assigned user 4. A CreateTask form component — title, description, priority, due date Rules: - Tasks belong to a project - Only project members can create/edit tasks - Status can be: TODO, IN_PROGRESS, DONE - Priority can be: LOW, MEDIUM, HIGH - Use server actions OR API routes (your choice, recommend API routes for beginners)
3
Test the full CRUD flow
Test each action manually
Create a new project → it appears in the project list
Click into a project → add a task → it appears in the project
Edit a task → changes are saved
Delete a task → it disappears
4
Commit to GitHub
Terminal
$ git add .
$ git commit -m "feat: add project and task CRUD"
$ git push
Phase 2 Complete — Projects and tasks work. Data saves to database.

Go to resend.com → Create free account → Get API key → Add to .env
.env — add this line
RESEND_API_KEY="re_xxxxxxxx"
Terminal
$ npm install resend
2
Add team invite + email notification
📋 Paste into Claude Code Chat
Add team collaboration features to my Task Tracking app. I need: 1. Invite a user to a project by email - /api/projects/[id]/members/route.ts — POST to add a member - Simple form: input email → click "Add member" - Look up user by email in the database, then add them to ProjectMember table 2. PHASE 6

Deploy to Production

By the end of Phase 6 — app is live on the internet. Anyone can use it from any device.
1
Deploy PostgreSQL on Railway
1. Go to railway.com → Sign up with GitHub
2. Click "New Project" → "Deploy PostgreSQL"
3. After it deploys, click the PostgreSQL service → "Variables" tab
4. Copy the DATABASE_URL value — you'll need it in the next step
ℹ️Railway gives you $5 free credit per month. A small PostgreSQL + Next.js app uses about $1-2/month.
2
Deploy Backend on Railway
1. In Railway, click "New" → "GitHub Repo" → Select your repo
2. Railway will auto-detect Next.js and start building
3. Go to "Variables" tab and add these:
Railway Environment Variables
DATABASE_URL = [paste from PostgreSQL service]
NEXTAUTH_SECRET = [any long random string]
NEXTAUTH_URL = https://your-app-name.up.railway.app
RESEND_API_KEY = [your Resend key]
Run database migration on production
# In Railway → your app service → Settings → Add this to "Start Command"
npx prisma migrate deploy && npm start
3
Deploy Frontend on Vercel
1. Go to vercel.com → Sign up with GitHub
2. Click "New Project" → Import your GitHub repo
3. Add the same environment variables as Railway
4. Click "Deploy"
5. Wait 2-3 minutes → your app is live!
🎉Your app is live! Share the Vercel URL with someone. Ask them to register and create a task. That's your first real user.
4
Final deploy checklist
📋 Paste into Claude Code Chat before going live
I'm about to deploy my Task Tracking app to production. Please give me a final checklist to verify everything is working. My stack: Next.js + PostgreSQL on Railway + Vercel Check: auth flow, task CRUD, kanban, email notification, environment variables
  • Register a new account on the live URL
  • Create a project and add tasks
  • Drag a task between columns
  • Invite a team member by email
  • Assign a task — check that email notification arrives
  • Test on mobile phone — does it look ok?
  • Share link with at least 1 real person
P01 Complete — Task Tracking System is live. Real users can use it. Move to P02.
