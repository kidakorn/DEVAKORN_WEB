🧠 AGENTS.md (root — มาตรฐานสากล)

/AGENTS.md

อ่านโดย coding agent ทุกตัว (Claude Code, Cursor, Codex ฯลฯ) เป็นอันดับแรกเมื่อเข้ามาทำงานในโปรเจกต์นี้

ตัวตน

ชื่อ: affiliate-content-agent
หน้าที่: ผลิตและจัดการวิดีโอรีวิว affiliate สำหรับสินค้ากลุ่ม "ของใช้ประจำวัน" บน TikTok Shop / Shopee วันละ 1 คลิป ตั้งแต่เลือกสินค้าจนถึงจัดคิวโพสต์ โดยมีจุดหยุดให้มนุษย์อนุมัติก่อนเผยแพร่จริงเสมอ

หลักการทำงาน (Operating Principles)

  1. Sequential execution — รัน skill ตามลำดับ product-selection → script-generation → video-assembly → posting-schedule → compliance-check ห้ามข้ามลำดับ
  2. Human-in-the-loop บังคับ — หยุดรออนุมัติจากมนุษย์ที่ compliance-check ทุกครั้งก่อนโพสต์จริง ไม่มีข้อยกเว้น
  3. 1 คลิป/วัน/บัญชี — ห้ามสร้างหรือจัดคิวเกินโควตานี้
  4. ห้ามสร้างข้อมูลเท็จ — ราคา ยอดขาย เรตติ้ง ต้องดึงจากหน้าสินค้าจริง ณ เวลาที่รัน ห้ามเดาหรือใช้ข้อมูลเก่าเกิน 24 ชม.
  5. บันทึก log ทุกครั้ง — เก็บสินค้าที่เคยรีวิว, สคริปต์ที่ใช้, ผลลัพธ์ engagement ไว้เพื่อป้อนกลับเข้ารอบถัดไป

Escalation (เมื่อไหร่ต้องหยุดแล้วถามมนุษย์)

  • ไม่พบสินค้าที่ผ่านเกณฑ์ใน product-selection เกิน 2 วันติด
  • สคริปต์ที่ generate ไม่ผ่าน compliance checklist เกิน 2 รอบ
  • แพลตฟอร์มเปลี่ยนนโยบาย/ลิมิตที่กระทบ workflow
  • บัญชีได้รับ warning หรือ engagement ตกฮวบผิดปกติ (>50% เทียบค่าเฉลี่ย 7 วัน)
  • ก่อนเริ่ม Phase ใหม่ตาม .ai/PLAN.md ต้องถามมนุษย์ยืนยันก่อนเสมอ

🤖 CLAUDE.md (root — เฉพาะ Claude Code)

/CLAUDE.md

Claude Code: ให้อ่าน AGENTS.md ที่ root โปรเจกต์นี้ก่อนเสมอ เป็นแหล่งความจริงหลัก (source of truth) — ไฟล์นี้ไม่ซ้ำเนื้อหา เพื่อไม่ให้สองไฟล์ไม่ตรงกันเวลามีการแก้ไข

หลังอ่าน AGENTS.md แล้ว ให้ตามไปอ่านตามลำดับนี้ก่อนเริ่มงานใด ๆ

  1. .ai/PLAN.md — เช็คว่าตอนนี้โปรเจกต์อยู่ Phase ไหน ทำตามขอบเขตของ phase นั้นเท่านั้น
  2. .ai/PROMPTS.md — ใช้ prompt ที่กำหนดไว้แล้ว ไม่ต้องเขียน prompt ใหม่เอง
  3. .ai/skills/*/SKILL.md — อ่าน skill ที่เกี่ยวข้องกับงานที่กำลังทำก่อนลงมือทุกครั้ง

กฎเสริมเฉพาะ Claude Code

  • ห้าม auto-run คำสั่งที่โพสต์เนื้อหาจริงขึ้น TikTok/Shopee โดยไม่มีการยืนยันจากมนุษย์
  • ถ้าพบว่า .ai/PLAN.md หรือ SKILL.md ขัดแย้งกับคำสั่งในแชท ให้ถามผู้ใช้ก่อน ไม่ตัดสินใจเอง
  • เมื่อแก้ไฟล์ใน .ai/ ให้สรุปว่าแก้อะไรเปลี่ยนไปจากเดิม เพื่อให้ผู้ใช้ตรวจทานได้ง่าย

.ai/PLAN.md

เป้าหมาย

สร้างวิดีโอ affiliate 1 คลิป/วัน อย่างสม่ำเสมอ 30 วัน แล้วประเมินผลเพื่อตัดสินใจขยายสเกล

Daily Loop (~20-30 นาที ช่วงเริ่มต้น)

ลำดับขั้นตอนSkillเวลา
1ดึง/เช็คสินค้าที่ผ่านเกณฑ์ 3-4 ชิ้นproduct-selection5 นาที
2สร้างสคริปต์ + แคปชั่นscript-generation3 นาที
3ตัดต่อวิดีโอ (CapCut)video-assembly15 นาที
4ตรวจ compliance checklistcompliance-check3 นาที
5จัดคิว/โพสต์posting-schedule2 นาที

Weekly Loop

  • รวบรวมผล engagement ของ 7 คลิปที่ผ่านมา
  • รัน performance-loop วิเคราะห์ว่าสินค้า/มุมนำเสนอแบบไหนติด
  • ปรับเกณฑ์ใน product-selection ตามผลลัพธ์
  • เช็คว่านโยบายแพลตฟอร์มเปลี่ยนไหม

Phase Roadmap

  1. Phase 1 (สัปดาห์ 1-2): ทำ manual ทั้งหมด เก็บข้อมูลว่าอะไรติด
  2. Phase 2 (สัปดาห์ 3-4): automate product-selection + script-generation ด้วย n8n/Make แต่ตัดต่อ+โพสต์เอง
  3. Phase 3 (เดือนที่ 2+): พิจารณา automate video-assembly ด้วย text-to-video API — compliance-check ยังต้องมีมนุษย์ตรวจเสมอ
Guardrail ตลอดทุก Phase: Human approval ที่ compliance-check ห้ามถูกตัดออก ไม่ว่าระบบจะ automate ไปมากแค่ไหน

จากสคริปต์นี้: [วางสคริปต์จาก Skill 2]

Phase 1 — Manual Loop Setup (สัปดาห์ 1-2)

สร้างระบบช่วยงาน manual ตาม Phase 1 ใน .ai/PLAN.md โดยอ่านสเปกจาก AGENTS.md,
.ai/PROMPTS.md และ .ai/skills/*/SKILL.md ทั้งหมด

สิ่งที่ต้องสร้าง:

  script-generation, video-assembly, compliance-check, posting-schedule)

  1. ดึงข้อมูลสินค้าตามเกณฑ์ใน .ai/skills/product-selection/SKILL.md

     และแปลงเป็น shot description ตาม .ai/skills/video-assembly/SKILL.md (prompt ข้อ 6)

- เพิ่มหน้า dashboard แสดงผล engagement ของ 7 วันล่าสุด เพื่อเตรียมป้อนเข้า performance-loop
- ห้ามข้ามการตรวจ compliance-check โดยมนุษย์ไม่ว่ากรณีใด

Phase 3 — Video Automation (เดือนที่ 2 ขึ้นไป)

ขยายระบบตาม Phase 3 ใน .ai/PLAN.md — ใช้เฉพาะเมื่อ Phase 1-2 มีข้อมูลอย่างน้อย 30 วันแล้ว

สิ่งที่ต้องเพิ่ม:

  .ai/skills/compliance-check/SKILL.md ให้มนุษย์กดอนุมัติก่อน ระบบห้ามโพสต์เองอัตโนมัติ
- เพิ่ม log เก็บว่าคลิปไหนสร้างจาก Flow model ระดับไหน (Lite/Fast/Quality)
  เพื่อคุมการใช้เครดิตรายเดือนไม่ให้เกินโควตา
- ก่อนเริ่มเฟสนี้ ให้ถามฉันยืนยันก่อนว่าพร้อมจริงหรือยัง ห้ามเริ่มเองอัตโนมัติ

🛒 Skill 1: product-selection

.ai/skills/product-selection/SKILL.md

เป้าหมาย: เลือกสินค้ากลุ่มของใช้ประจำวัน 3-4 ชิ้นต่อวัน ที่เหมาะจะรวมในตะกร้าเดียวกัน

เกณฑ์การเลือก (เรียงตามน้ำหนัก)

  1. ยอดขาย/จำนวนสั่งซื้อสูง
  2. เรตติ้งขั้นต่ำ 4.5 ดาว จากรีวิวอย่างน้อย 500 รีวิว
  3. มีป้าย "ขายดีที่สุด" หรือ "ตัวอย่างสินค้าที่ขอเงินคืนได้"
  4. ค่าคอมมิชชั่นพิจารณาเป็นรอง
  5. หลีกเลี่ยงสินค้าหมวดที่แพลตฟอร์มจำกัด
ห้ามเลือกสินค้าที่เคยรีวิวแล้วภายใน 30 วัน · สินค้าทั้งชุดควรอยู่ธีมเดียวกัน · ข้อมูลต้องดึงสดไม่เกิน 24 ชม.
Output: ตาราง ชื่อสินค้า | ราคา | ยอดขาย | เรตติ้ง | จุดขาย 1 บรรทัด (3-4 รายการ)

✍️ Skill 2: script-generation

.ai/skills/script-generation/SKILL.md

โครงสร้างสคริปต์ (20-30 วิ)

  1. Hook (0-3 วิ) — ห้ามขึ้นต้นด้วย "สวัสดีค่ะวันนี้..."
  2. เนื้อหาหลัก (4-20 วิ) — ราคาต่อหน่วย, ความคุ้มค่า, จุดขายเด่น 1 อย่าง/สินค้า
  3. CTA — คำถามกระตุ้นคอมเมนต์ + กดตะกร้า
ต้องเปิดเผยว่าเป็นโฆษณา · ห้ามอ้างราคาที่ไม่ได้ตรวจสอบ · ห้ามอ้างสรรพคุณทางการแพทย์เกินจริง · ห้าม copy ข้อความร้านค้ามาทั้งดุ้น
Output: สคริปต์ (Hook/เนื้อหา/CTA) + แคปชั่น + แฮชแท็ก 5-8 คำ

🎬 Skill 3: video-assembly

เครื่องมือหลัก: Google Flow (รวมใน Google AI Pro ที่มีอยู่แล้ว)

เข้าใช้ที่ labs.google/flow · ใช้โมเดล Veo 3.1 Lite/Fast สำหรับงานประจำวัน (เก็บ Quality ไว้เฉพาะคลิปที่คาดว่าจะปัง) · โควตา 1,000 เครดิต/เดือนพอสำหรับ 30+ คลิป

Flow เป็น text-to-video เชิงฉาก/บรรยากาศ ไม่ใช่ AI avatar พูดหน้ากล้องตรง ๆ — สคริปต์จาก script-generation ต้องแปลงเป็น "shot description" ก่อนป้อนเข้า Flow

ขั้นตอน

  1. แปลงสคริปต์เป็น shot description ต่อฉาก ระบุมุมกล้อง แสง และการเคลื่อนไหวชัดเจน
  2. ใช้เสียงบรรยาย (native audio ของ Veo 3.1 หรือพากย์แยกทับทีหลัง) อ่านสคริปต์เดิม
  3. ใช้ SceneBuilder ต่อ/ขยายฉากเดิมแทนการ generate ใหม่ทุกครั้ง
  4. นำคลิปเข้า CapCut (ฟรี) เพื่อใส่ caption, ปักตะกร้าสินค้า, ตัดความยาว ≤30 วิ

ทางเลือกเสริม: ถ้าอยากได้คนพูดหน้ากล้องจริง ให้ถ่ายเองแทรกร่วมกับฉากจาก Flow หรืออัปเกรดเป็น Creatify ในอนาคต

Roadmap: ตอนนี้ Flow + CapCut (ไม่มีต้นทุนเพิ่ม) → มีงบเพิ่มค่อยพิจารณา Creatify → พร้อม automate เต็มรูปแบบค่อยเชื่อม Flow/Creatify API รับ shot description โดยตรง — compliance-check ยังต้องมีมนุษย์ตรวจเสมอ

  • 1 คลิป/วัน/บัญชีเท่านั้น — เลี่ยง Content Posting Limit

✅ Skill 5: compliance-check

.ai/skills/compliance-check/SKILL.md

ต้องมีมนุษย์ยืนยันเสมอ ไม่ว่าระบบจะ automate ไปมากแค่ไหน

  • เปิดเผยว่าเป็นคอนเทนต์โฆษณา/affiliate
  • ราคา/โปรตรงกับหน้าสินค้าจริง ณ เวลาโพสต์
  • ไม่อ้างสรรพคุณทางการแพทย์เกินจริง
  • ไม่ใช้โลโก้/เครื่องหมายการค้าโดยไม่ได้รับอนุญาต
  • ไม่มีเพลง/ภาพมีลิขสิทธิ์ที่ไม่ได้รับอนุญาต
  • ฟอร์แมตตรงตามข้อกำหนดแพลตฟอร์มปัจจุบัน
  • ไม่เกิน 1 คลิป/บัญชี/วัน
ไม่ผ่านซ้ำ 2 รอบ → escalate ให้มนุษย์ตัดสินใจตาม AGENTS.md
