2Backend — AS400 + IWS ส่วนของแอดมิน

CREATE TABLE ERPLIB.STOCK_ITEMS (
  ITEM_CODE     CHAR(10)     NOT NULL,
  ITEM_NAME     VARCHAR(100),
  CATEGORY      VARCHAR(30),
  UNIT          VARCHAR(10),
  QTY_ON_HAND   DECIMAL(10,2) DEFAULT 0,
  REORDER_POINT DECIMAL(10,2) DEFAULT 0,
  PRIMARY KEY (ITEM_CODE)
);

CREATE TABLE ERPLIB.APP_USERS (
  USER_ID    CHAR(10)    NOT NULL,
  FULL_NAME  VARCHAR(80),
  DEPARTMENT VARCHAR(50),
  ROLE       VARCHAR(20),
  STATUS     CHAR(1) DEFAULT 'A',
  PRIMARY KEY (USER_ID)
);

Endpoint ที่ควรขอแอดมิน deploy (ผ่าน IWS Studio → Deploy New Service → SQL statement):

EndpointMethodคำอธิบาย
/stock/GET_ITEMSGETดึงรายการสินค้า+จำนวนคงเหลือ
/stock/STOCK_INPOSTรับสินค้าเข้า (insert trans + update qty)
/stock/STOCK_OUTPOSTเบิกสินค้าออก
/user/GET_USERSGETดึงรายชื่อผู้ใช้
/user/ADD_USERPOSTเพิ่มผู้ใช้ใหม่
/user/SET_STATUSPOSTเปิด/ปิดการใช้งาน user
/po/GET_POGETดึงรายการใบสั่งซื้อ
/po/CREATE_POPOSTสร้าง PO ใหม่ (header+detail)
/po/RECEIVE_POPOSTรับของตาม PO → stock บวกอัตโนมัติ
/so/GET_SOGETดึงรายการใบสั่งขาย
/so/CREATE_SOPOSTสร้าง SO ใหม่ → stock ลบอัตโนมัติ
ทำไมต้องแยก RECEIVE_PO ออกจาก CREATE_PO

สั่งซื้อ (CREATE_PO) กับของมาถึงจริง (RECEIVE_PO) มักไม่ใช่วันเดียวกัน — แยก endpoint ไว้ stock จะได้บวกตอนของมาถึงจริงเท่านั้น ไม่ใช่ตอนสั่งซื้อ

4js/config.js

const ENVIRONMENTS = {
  local: { BASE_URL: "/api/web/serveices" },
  production: { BASE_URL: "/api/web/serveices" }
};

function detectEnvironment() {
  const host = window.location.hostname;
  return (host === "localhost" || host === "127.0.0.1") ? "local" : "production";
}

const IWS_CONFIG = {
  BASE_URL: ENVIRONMENTS[detectEnvironment()].BASE_URL,
  ENDPOINTS: {
    GET_ITEMS:  "/stock/GET_ITEMS",
    STOCK_IN:   "/stock/STOCK_IN",
    STOCK_OUT:  "/stock/STOCK_OUT",
    GET_USERS:  "/user/GET_USERS",
    ADD_USER:   "/user/ADD_USER",
    SET_STATUS: "/user/SET_STATUS",
    GET_PO:     "/po/GET_PO",
    CREATE_PO:  "/po/CREATE_PO",
    GET_SO:     "/so/GET_SO",
    CREATE_SO:  "/so/CREATE_SO"
  }
};

5js/api.js

async function handleResponse(res) {
  const raw = await res.text();
  if (!res.ok) throw new Error(`HTTP ${res.status}: ${raw}`);
  return JSON.parse(raw);
}
function post(endpoint, payload) {
  return fetch(IWS_CONFIG.BASE_URL + endpoint, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(payload)
  }).then(handleResponse);
}

const STOCK_API = {
  getItems:  () => fetch(IWS_CONFIG.BASE_URL + IWS_CONFIG.ENDPOINTS.GET_ITEMS).then(handleResponse),
  stockIn:   (p) => post(IWS_CONFIG.ENDPOINTS.STOCK_IN, p),
  stockOut:  (p) => post(IWS_CONFIG.ENDPOINTS.STOCK_OUT, p)
};
const USER_API = {
  getUsers:  () => fetch(IWS_CONFIG.BASE_URL + IWS_CONFIG.ENDPOINTS.GET_USERS).then(handleResponse),
  addUser:   (p) => post(IWS_CONFIG.ENDPOINTS.ADD_USER, p),
  setStatus: (p) => post(IWS_CONFIG.ENDPOINTS.SET_STATUS, p)
};
const PO_API = {
  getPO:    () => fetch(IWS_CONFIG.BASE_URL + IWS_CONFIG.ENDPOINTS.GET_PO).then(handleResponse),
  createPO: (p) => post(IWS_CONFIG.ENDPOINTS.CREATE_PO, p)
};
const SO_API = {
  getSO:    () => fetch(IWS_CONFIG.BASE_URL + IWS_CONFIG.ENDPOINTS.GET_SO).then(handleResponse),
  createSO: (p) => post(IWS_CONFIG.ENDPOINTS.CREATE_SO, p)
};

8Spec ส่งแอดมิน

ขอ deploy IWS service สำหรับระบบ Stock & User ครับ

STOCK
1) GET_ITEMS (GET)   → ITEM_CODE, ITEM_NAME, CATEGORY, UNIT, QTY_ON_HAND, REORDER_POINT
   Source: ERPLIB.STOCK_ITEMS
2) STOCK_IN (POST)   Input: ITEM_CODE, QTY, REMARK  → insert STOCK_TRANS + update QTY_ON_HAND (+)
3) STOCK_OUT (POST)  Input: ITEM_CODE, QTY, REMARK  → insert STOCK_TRANS + update QTY_ON_HAND (-)

USER
4) GET_USERS (GET)   → USER_ID, FULL_NAME, DEPARTMENT, ROLE, STATUS
   Source: ERPLIB.APP_USERS
5) ADD_USER (POST)   Input: USER_ID, FULL_NAME, DEPARTMENT, ROLE
6) SET_STATUS (POST) Input: USER_ID, STATUS (A/I)

PURCHASE ORDER
7) GET_PO (GET)      → PO_NO, SUPPLIER, PO_DATE, STATUS, ITEM_CODE, QTY, UNIT_PRICE
   Source: ERPLIB.PO_HEADER join PO_DETAIL
8) CREATE_PO (POST)  Input: SUPPLIER, ITEM_CODE, QTY, UNIT_PRICE → insert PO_HEADER+DETAIL, STATUS='O'
9) RECEIVE_PO (POST) Input: PO_NO → update STATUS='R' + update STOCK_ITEMS.QTY_ON_HAND (+)

SALES ORDER
10) GET_SO (GET)     → SO_NO, CUSTOMER, SO_DATE, STATUS, ITEM_CODE, QTY, UNIT_PRICE
    Source: ERPLIB.SO_HEADER join SO_DETAIL
11) CREATE_SO (POST) Input: CUSTOMER, ITEM_CODE, QTY, UNIT_PRICE → insert SO_HEADER+DETAIL, update STOCK_ITEMS.QTY_ON_HAND (-)

รบกวนแจ้ง URL/path ของแต่ละ endpoint หลัง deploy เสร็จด้วยครับ ขอบคุณครับ
