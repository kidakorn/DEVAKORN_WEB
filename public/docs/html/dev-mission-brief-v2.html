01

Mindset: Developer + AI Agent

การทำงานกับ AI ไม่ใช่แค่ถาม-ตอบ แต่คือการทำงานเป็นทีมกับ AI ที่รู้ทุกอย่าง แต่ไม่รู้ context ของคุณ — หน้าที่คุณคือ brief ให้ชัด ตัดสินใจเอง และ verify ทุกอย่าง

// Phase 01 — ตอนนี้
Foundation
เดือน 1–3
  • เรียนรู้ full-stack ด้วยการทำจริง
  • สร้าง 2–3 โปรเจ็คเล็กให้ deploy ได้
  • ฝึก brief AI ให้ถูกต้อง
  • เรียนรู้ error จากการแก้จริง
// Phase 02
First Product
เดือน 3–6
  • เลือก 1 niche สร้าง MVP จริง
  • หาลูกค้าแรกก่อน launch
  • Deploy + support จริง
  • เพิ่ม AI feature แรก
// Phase 03
SaaS + AI Scale
เดือน 6+
  • Multi-tenant รองรับหลาย org
  • AI Agent ทำงานแทนพนักงาน
  • Recurring revenue สม่ำเสมอ
  • พัฒนาโปรเจ็คถัดไปคู่กัน
⚡ กฎทอง: อย่าวางแผนนานเกิน 1 สัปดาห์ก่อนลงมือ ทุกสิ่งที่คุณไม่รู้จะเจอได้แค่ตอนทำจริงเท่านั้น ใช้ AI ช่วยแก้ปัญหาที่เจอจริง ไม่ใช่วางแผนปัญหาที่ยังไม่เจอ

CLAUDE CODE — ระหว่างพัฒนา ติดปัญหา
.build
Task: สร้าง API endpoint POST /api/leave-requests
Context: ใช้ Next.js App Router, Prisma, user login ด้วย NextAuth
Schema มีแล้ว: LeaveRequest { id, userId, type, startDate, endDate, status }
ปัญหา: ไม่รู้จะ validate date range ยังไง และ deduct quota ยังไง
SOLUTION ช่วยเขียน API route พร้อม validation + quota check
→ validate: endDate > startDate, ไม่ซ้อน leave เดิม
→ quota: นับวันใน DB ก่อน เช็คว่าเกิน annual_leave_days ไหม
→ transaction: create leave + update quota ใน DB transaction เดียว
💡 บอก context ให้ครบ: framework, schema, และปัญหาที่ติด — อย่าถามแค่ "สร้าง API ยังไง"
CLAUDE CODE — มี error แก้ไม่ออก
.debug
Error: PrismaClientKnownRequestError: Foreign key constraint failed
เกิดตอน: create LeaveRequest
Code: [วาง code ที่ error]
Schema: [วาง schema ที่เกี่ยวข้อง]
ROOT CAUSE userId ที่ส่งมาไม่มีใน User table
FIX ดึง session.user.id จาก NextAuth ก่อน create
→ อย่าเชื่อ userId จาก request body โดยตรง (security issue ด้วย)
💡 ส่ง error message + code + schema ครบ — อย่าแค่ถามว่า "ทำไม error"
CLAUDE CODE — ก่อน deploy ทุกครั้ง
.review
ช่วย review code นี้ก่อน deploy
โฟกัส: Security, Performance, Error handling
[วาง code ทั้ง file หรือ PR diff]
SECURITY ❌ ไม่มี rate limiting บน auth endpoints
SECURITY ❌ error message บอก "user not found" — ควรบอกแค่ "invalid credentials"
PERF ⚠️ query ใน loop — ควร batch query แทน
OK Input validation, Auth middleware, Error handling ผ่าน
💡 ใช้ .review ก่อน deploy ทุกครั้ง — AI จะจับ security issue ที่คุณอาจมองข้าม

เพิ่ม AI Agent Layer

เมื่อระบบพร้อม — นี่คือวิธีเพิ่ม AI เข้าไปในแต่ละโปรเจ็คที่ผ่านมา ให้มันทำงานแทนมนุษย์จริงๆ

AI AGENT PATTERN — ใช้กับทุกโปรเจ็ค
.agent สรุปรายงานธุรกิจประจำวัน
ดึงข้อมูล: ยอดขาย, task ค้าง, stock ใกล้หมด, ticket รอตอบ
ส่ง: LINE Notify ทุก 8:00 น.
Format: สรุปสั้น 5 บรรทัด อ่านใน 30 วินาที
STACK Cron job (node-cron) → Query DB → Claude API → LINE Notify

PATTERN
1. Cron รัน 08:00 ทุกวัน
2. Query: ยอดขายเมื่อวาน, task ค้าง 3 วัน, stock < reorder point
3. ส่งข้อมูลให้ Claude API: "สรุปข้อมูลนี้เป็นรายงานผู้บริหาร 5 บรรทัด"
4. ส่ง response → LINE Notify / Email

Claude API model: claude-sonnet-4-6, max_tokens: 500
💡 AI Agent ที่ดีที่สุดคือตัวที่ "ทำงานเงียบๆ แล้วแจ้งเมื่อต้องการคน" ไม่ใช่ตัวที่ถามทุกอย่าง
🤖 Agent: Auto Report
Cron → Query DB → Claude API → สรุป → LINE/Email ทุกเช้า ใช้กับ CRM, WMS, POS
🤖 Agent: Smart Reply
Ticket เข้า → Claude draft คำตอบ → เจ้าหน้าที่ approve 1 click → ส่ง ใช้กับ Helpdesk
🤖 Agent: Data Entry
Upload ใบเสร็จ/เอกสาร → Claude OCR + extract → กรอก form อัตโนมัติ ใช้กับ Accounting
🤖 Agent: Forecast
ดึงข้อมูลย้อนหลัง → Claude วิเคราะห์แนวโน้ม → alert ล่วงหน้า ใช้กับ WMS, CRM
