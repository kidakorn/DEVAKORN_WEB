IBM i Integration · Environment Comparison

ทำที่ บริษัท vs ทำบน PC ตัวเอง

เทียบทุก solution ว่าวิธีไหนทำได้จริงในแต่ละสภาพแวดล้อม

1 · ODBC / JDBC

🔌

ODBC / JDBC Driver

Node.js query DB2 ตรงผ่าน driver
ทำได้ ✓
🏢 บริษัท (AS/400 จริง)
Node.js→ ODBC (LAN)→ DB2 บริษัท
Latency ต่ำมาก (LAN) — เร็วและเสถียรที่สุด ใช้ production ได้เต็มรูปแบบ
ทำได้ ✓
💻 PC ตัวเอง (PUB400)
Node.js→ ODBC (Internet)→ PUB400
Latency สูงกว่า (ข้ามประเทศ) — ใช้ฝึกได้เต็มที่ แต่ไม่เหมาะ production

2 · IBM i Web Services (IWS)

🏭

IWS — Expose RPG/CL เป็น REST อัตโนมัติ

ต้องติดตั้ง + config โดย Admin ระดับ OS
ขึ้นกับ Admin
🏢 บริษัท (AS/400 จริง)
RPG/CL→ IWS Service→ REST API
ทำได้ถ้า Admin เปิดสิทธิ์ติดตั้ง/start HTTP instance — เป็นเครื่องของบริษัทเองจึงมีโอกาสมากกว่า
ทำไม่ได้ ✗
💻 PC ตัวเอง (PUB400)
RPG/CL✗ IWS
PUB400 เป็น shared server — user ทั่วไปไม่มีสิทธิ์ start/config HTTP Server หรือเปิด port

3 · itoolkit + Node.js (Call RPG ผ่าน SSH/XMLSERVICE)

🛠

itoolkit — เขียน REST เองทั้งหมด

Node.js เรียก RPG/CL ผ่าน SSH + XMLSERVICE
ทำได้ ✓
🏢 บริษัท (AS/400 จริง)
Express→ SSH (LAN)→ XMLSERVICE→ RPG
SSH ภายใน LAN เร็ว overhead น้อย — ใช้งานจริงได้สบาย
ทำได้ ✓
💻 PC ตัวเอง (PUB400)
Express→ SSH (Internet)→ XMLSERVICE→ RPG
ทำได้เต็มที่ — XMLSERVICE มีอยู่แล้วบน PUB400 แต่ SSH ข้ามประเทศจะมี latency เพิ่ม (~100-300ms/call)

4 · File-based (CL Download/Upload → CSV/Excel)

📁

File-based Workflow

CL สร้างไฟล์ → Web อ่าน/เขียนไฟล์ → CL Upload กลับ
ทำได้ ✓
🏢 บริษัท (AS/400 จริง)
CL→ Shared Folder (LAN)→ Web App
Mount Shared Folder ผ่าน LAN ได้ตรง — Node.js FileWatcher ทำงานทันที
ทำได้ ✓
💻 PC ตัวเอง (PUB400)
CL → IFS→ SFTP Download→ Web App
ใช้ SFTP/SSH ดึงไฟล์จาก IFS มาที่ local แทน Shared Folder — ทำงานเหมือนกัน แค่ผ่าน Internet

5 · Db2 REST (5733-REST)

🔴

Db2 REST — IBM Licensed Service

Query รันบน IBM i เลย ผ่าน HTTP endpoint
ขึ้นกับ Admin + License
🏢 บริษัท (AS/400 จริง)
Browser→ HTTP (LAN)→ Db2 REST Service
ต้องซื้อ/ติดตั้ง License 5733-REST + IBM i 7.3+ และให้ Admin ติดตั้ง — เป็นไปได้เพราะเป็นเครื่องบริษัท
ทำไม่ได้ ✗
💻 PC ตัวเอง (PUB400)
Db2 REST✗
เป็น Licensed Program ที่ user ทั่วไปติดตั้งเองไม่ได้ และ PUB400 ไม่อนุญาต

6 · ODBC + Redis Cache

⚡

Cache Layer เพิ่มทับ ODBC

ลด round-trip ไป IBM i ด้วย in-memory/Redis cache
ทำได้ ✓
🏢 บริษัท (AS/400 จริง)
Request→ Redis→ ODBC (LAN)
ได้ผลดีมาก — ODBC เร็วอยู่แล้ว Cache ช่วยลด load DB2 ตอน peak hour
ทำได้ ✓
💻 PC ตัวเอง (PUB400)
Request→ Redis→ ODBC (Internet)
ได้ผลดีมากเช่นกัน — ช่วยชดเชย latency ของ PUB400 ได้ดีเพราะ cache hit ไม่ต้องข้าม Internet

สรุปภาพรวม

🎯 ข้อแตกต่างหลักระหว่าง 2 สภาพแวดล้อม

01 — LATENCY

บริษัท = LAN (ms เดียว) ส่วน PUB400 = ข้าม Internet/ต่างประเทศ (10-50x ช้ากว่า) ทุก solution ที่ "ทำได้ทั้งคู่" จะช้ากว่าบน PUB400 เสมอ

02 — ADMIN RIGHTS

IWS และ Db2 REST ต้องการ Admin ติดตั้ง licensed program — เป็นไปได้ที่บริษัท (เครื่องตัวเอง) แต่ปิดสนิทบน PUB400 (shared server)

03 — ตัวที่ใช้ฝึกได้เต็มที่

ODBC, itoolkit, File-based, และ Redis Cache ทำได้ "เหมือนกันทุกอย่าง" ทั้ง 2 ที่ — เหมาะที่สุดสำหรับ build skill ตอนนี้บน PUB400

04 — ย้ายไปบริษัทแล้วใช้ต่อได้

Code ที่เขียนบน PUB400 ด้วย ODBC/itoolkit/Redis ย้ายไปบริษัทได้ทันที — แค่เปลี่ยน connection string จาก PUB400 → AS/400 บริษัท
