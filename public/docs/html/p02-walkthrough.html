An HR leave management app for small-to-medium teams. Employees submit

## Coding Rules
- Use **TypeScript** for all files
- Use **async/await**
- Always validate input server-side before saving to database
- Never trust data from the frontend — verify role AND ownership on every request
- Use Prisma for ALL database queries
- Commit to GitHub after every completed feature
- Never hardcode user-facing text — use next-intl translation keys
- Comment every new or changed piece of code
- **Every API route that touches LeaveRequest, LeaveQuota, or User must check
  role + ownership before returning data — this is HR data, treat it as sensitive by default**
