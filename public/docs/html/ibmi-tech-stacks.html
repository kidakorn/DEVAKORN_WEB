1 · .NET / C#

🟣

.NET (C#) — IBM i Access Client Provider

Official .NET Data Provider จาก IBM เร็วกว่า ODBC ในหลายกรณี และมี Program Call API เรียก RPG ตรงๆได้
Official IBM Driver ตลาดงานไทยกว้าง

วิธีเชื่อม IBM i

ASP.NET Core→ IBM.Data.Db2.iSeries→ DB2 / RPG Program
IBM.Data.Db2.iSeries — SQL + Program Call
iDB2Connection — connection class
ASP.NET Core Web API — สร้าง REST endpoint
Entity Framework Core — ORM (optional)

แนวทางพัฒนา

1. ติดตั้ง IBM i Access Client Solutions (ACS) บนเครื่อง
2. ใช้ iDB2Connection connect ผ่าน connection string
3. Query ปกติด้วย iDB2Command
4. Call RPG Program ผ่าน Program Call class — ไม่ต้อง SSH/itoolkit

เหมาะกับ: สาย Enterprise / Windows Server / ต้องการ type-safety
เรียนรู้: C# syntax (~2-4 สัปดาห์ถ้าไม่มีพื้นฐาน OOP) + ASP.NET Core Web API (~2 สัปดาห์)

2 · PHP

🐘

PHP — ODBC / PDO_ODBC

Concept เดียวกับ Node.js ทุกอย่าง — ใช้ ODBC driver ตัวเดียวกัน เปลี่ยนแค่ภาษา
มีพื้นฐานแล้ว ง่ายสุดในการย้าย

วิธีเชื่อม IBM i

Laravel/PHP→ PDO_ODBC→ DB2 (PUB400)
PDO_ODBC — เหมือน node-odbc
odbc_connect() — native function
Laravel/Symfony — framework เร็วๆ

แนวทางพัฒนา

1. Connection string เดียวกับที่ใช้กับ Node.js
2. new PDO("odbc:DSN") → query เหมือนเดิม
3. Call SP: CALL X0250661.ISSUE_REQ(?)

เหมาะกับ: migrate ง่ายที่สุดเพราะ logic เหมือน Node.js 90%
เรียนรู้: แทบไม่ต้องเรียนใหม่ — แค่ syntax PHP ที่มีพื้นฐานอยู่แล้ว

3 · Angular (Frontend only)

🅰️

Angular — Frontend Framework

ไม่เชื่อม IBM i ตรง — เป็น Frontend ที่เรียก REST API (จาก Node.js/.NET/PHP) เหมือน Next.js
มีพื้นฐานแล้ว Frontend อย่างเดียว

ตำแหน่งใน Architecture

Angular→ HTTP/REST→ Backend (Node/.NET/PHP)→ IBM i
HttpClient — เรียก REST API
RxJS — handle async data
Angular Material — UI components

แนวทางพัฒนา

Angular ไม่ได้ "support IBM i" โดยตรง — มันคือ frontend เหมือน Next.js

ถ้าจะใช้ Angular แทน Next.js: Backend ยังเป็น Node.js/Express เดิม เปลี่ยนแค่ frontend

เหมาะกับ: ทีมที่คุ้น Angular / Enterprise UI ที่ต้องการ structure เข้มงวด
หมายเหตุ: ไม่กระทบ backend เลย — เปลี่ยน frontend ได้โดยไม่ต้องแก้ IBM i connection

4 · Java / Spring Boot

☕

Java — JT400 / JDBC

JT400 (IBM Toolbox for Java) เป็น library เก่าแก่ที่สุดและทรงพลังที่สุด — ครอบคลุมทุกอย่างของ IBM i
ครบเครื่องที่สุด Enterprise มาตรฐาน

วิธีเชื่อม IBM i

Spring Boot→ JT400.jar→ AS400 object
JT400 — official, มี ProgramCall, DataQueue, IFS API
AS400JDBCDriver — JDBC
Spring Data — repository pattern

แนวทางพัฒนา

JT400 ทำได้ทุกอย่างที่ itoolkit ทำ แต่ เร็วกว่าและไม่ต้อง SSH
ใช้ ProgramCall เรียก RPG ตรงผ่าน socket connection

เหมาะกับ: องค์กรใหญ่ที่มี Java team อยู่แล้ว
เรียนรู้: ใช้เวลานานสุดถ้าไม่มีพื้นฐาน Java/Spring (~1-2 เดือน)

5 · Python

🐍

Python — pyodbc / FastAPI

เหมาะกับงาน Data Analysis, Report, Automation จาก AS/400 data
เรียนง่าย เหมาะ Data/Report

วิธีเชื่อม IBM i

FastAPI→ pyodbc→ DB2 (PUB400)
pyodbc — เหมือน node-odbc
pandas — วิเคราะห์ข้อมูลที่ดึงมา
FastAPI — REST API เร็วและง่าย

แนวทางพัฒนา

Concept เดียวกับ Node.js ODBC ทุกอย่าง
จุดเด่นคือ pandas — ทำ data transform/report จาก AS/400 ได้ง่ายมาก

เหมาะกับ: งาน Report, Data Pipeline, Excel automation (แทน CL script บางส่วน)
เรียนรู้: เร็วถ้ามีพื้นฐานเขียนโค้ดอยู่แล้ว (~2-3 สัปดาห์)

6 · RPG Open Access / IWS (ฝั่ง IBM i เอง)

🔴

เขียนฝั่ง IBM i ให้ Expose เอง

ไม่ใช่ "เชื่อมจากภายนอก" แต่คือทำให้ IBM i เปิด API จากด้านใน — ต้องมี Admin
ต้องการ Admin

เครื่องมือ

RPG Open Access — RPG handler เขียน Web UI
IWS (5733-WS1) — Auto REST/SOAP
Db2 REST (5733-REST)
สถานะ: เหมือนที่คุยกันก่อนหน้า — ทำไม่ได้บน PUB400 เพราะต้องการ Admin ติดตั้ง Licensed Program
ใส่ไว้เพื่อความครบถ้วนของภาพรวมเท่านั้น
