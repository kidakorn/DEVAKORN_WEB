Variables & Types
C# เป็น strongly typed — ต้องระบุ type แต่ใช้ var ให้ compiler อนุมานได้ เหมือน TypeScript
C#
int age = 25; string name = "Kid"; bool isActive = true; var city = "Bangkok"; // อนุมาน type
TypeScript
let age: number = 25; let name: string = "Kid"; let isActive: boolean = true; let city = "Bangkok";
intstringbooldoublevardecimal
Functions & Lambda
C#
int Add(int a, int b) => a + b; var square = (int x) => x * x; async Task<string> Fetch() { await Task.Delay(1000); return "done"; }
TypeScript
const add = (a: number, b: number) => a + b; const square = (x: number) => x * x; async function fetch(): Promise<string> { await new Promise(r => setTimeout(r,1000)); return "done"; }
async/await ใน C# เหมือน JS แทบทุกอย่าง แค่เปลี่ยน Promise เป็น Task
Collections & LINQ
LINQ คือ Array methods ของ C# — คล้าย .filter() .map() .find() ใน JS
C# LINQ
var nums = new List<int> {1,2,3,4,5}; var evens = nums.Where(n => n % 2 == 0); var doubled = nums.Select(n => n * 2); var sum = nums.Sum(); var first = nums.FirstOrDefault(n => n > 3);
JavaScript
const nums = [1,2,3,4,5]; const evens = nums.filter(n => n % 2 === 0); const doubled = nums.map(n => n * 2); const sum = nums.reduce((a,b) => a+b, 0); const first = nums.find(n => n > 3);
List<T>Dictionary<K,V>WhereSelectFirstOrDefault
Null Handling
C#
string? name = null; // nullable type var len = name?.Length ?? 0; // null-coalescing เหมือน JS!
TypeScript
let name: string | null = null; const len = name?.length ?? 0; // เหมือนกันเลย!
Optional chaining ?. และ null-coalescing ?? ใช้ได้ใน C# เหมือนกันเลย

OOP & Design Patterns
2–3 เดือน · Class, Interface, MVVM
Class & Object
C#
public class Person { public string Name { get; set; } public int Age { get; set; } public Person(string name, int age) { Name = name; Age = age; } public string Greet() => $"สวัสดี ฉันชื่อ {Name}"; }
TypeScript
class Person { name: string; age: number; constructor(name: string, age: number) { this.name = name; this.age = age; } greet(): string { return `สวัสดี ฉันชื่อ ${this.name}`; } }
แทบเหมือน TypeScript เลย ต่างแค่ Property syntax กับ $"..." แทน template literal
Interface & Inheritance
C#
public interface IAnimal { string Sound(); } public class Dog : IAnimal { public string Sound() => "โฮ่ง!"; } public class Cat : IAnimal { public string Sound() => "เมี้ยว!"; }
TypeScript
interface IAnimal { sound(): string; } class Dog implements IAnimal { sound(): string { return "โฮ่ง!"; } } class Cat implements IAnimal { sound(): string { return "เมี้ยว!"; } }
