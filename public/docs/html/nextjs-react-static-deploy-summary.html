webas400app / kidakorn

Next.js / React — ใช้กับ Shared Folder ได้ไหม

สรุปว่า framework ไหนใช้ได้ ใช้ไม่ได้ กับโฟลเดอร์แชร์ที่เป็น static hosting (IIS ไม่มี Node.js runtime)

หลักการเดียวที่ต้องจำ: โฟลเดอร์แชร์ serve ได้แค่ไฟล์ HTML/CSS/JS ที่ build เสร็จแล้ว (static) เท่านั้น — ไม่มี Node.js รันอยู่เบื้องหลัง ดังนั้น framework ที่ใช้ได้ต้องเป็นแบบที่ build ออกมาเป็นไฟล์ static ล้วนๆ

สรุปเปรียบเทียบ

Framework / Featureใช้ได้ไหมหมายเหตุ
React (Vite / CRA)✅ ใช้ได้build แล้วได้ static files ล้วนๆ
Vue / Svelte / Angular✅ ใช้ได้หลักการเดียวกับ React
Next.js (Static Export)✅ ใช้ได้ต้องตั้ง output: 'export'
Next.js API Routes❌ ใช้ไม่ได้ต้องมี Node.js server รันตลอดเวลา
Next.js SSR / Middleware / ISR❌ ใช้ไม่ได้ต้องการ server เช่นกัน
Client-side fetch() เรียก IWS✅ ใช้ได้เกิดในเบราว์เซอร์ ไม่เกี่ยวกับ server

▲ Next.js — ใช้ได้แค่โหมด Static Export

ต้องตั้งค่าก่อน build ไม่งั้นจะได้ผลลัพธ์ที่ใช้กับ static hosting ไม่ได้

// next.config.js
module.exports = {
  output: 'export'
}
npx next build

จะได้โฟลเดอร์ out/ — เอาไฟล์ข้างในไปวางในโฟลเดอร์แชร์ได้เลย

🔴 ข้อจำกัด: ห้ามใช้ API Routes, Server Components ที่ fetch จาก server, Middleware, หรือ Image Optimization (ต้องปิดด้วย images: { unoptimized: true }) — ใช้ได้แค่ Client Component ที่เรียก fetch() ไปหา IWS ตรงๆ เหมือนที่ทำกันอยู่

⚛️ React (Vite) — ใช้ได้ตรงไปตรงมา

ไม่มีข้อจำกัดพิเศษ เพราะ React เป็น client-side ล้วนอยู่แล้ว

npm create vite@latest my-app -- --template react
cd my-app
npm install
npm run build

จะได้โฟลเดอร์ dist/ — เอาไฟล์ข้างในไปวางในโฟลเดอร์แชร์ได้เลย

Flow การใช้งานจริง (ทั้งสองแบบเหมือนกัน)

  • 1เขียนโค้ด + build บนเครื่องตัวเอง (มี Node.js/terminal อยู่แล้ว)
  • 2ได้โฟลเดอร์ output (out/ สำหรับ Next.js หรือ dist/ สำหรับ React/Vite)
  • 3Copy ไฟล์ข้างในโฟลเดอร์นั้นไปวางใน webas400app/kidakorn/
  • 4เว็บทำงานเหมือนเดิม เรียก IWS ผ่าน relative path (/api/web/services/...)

เรื่อง web.config ที่ต้องรู้เพิ่ม

รูปแบบแอปต้องทำอะไรกับ web.config
Next.js Static Export (หลายหน้า .html แยกกัน)ไม่ต้องเพิ่ม SPA fallback — แต่ละหน้ามีไฟล์ .html จริงอยู่แล้ว
React ที่ใช้ React Router (SPA เดียว)ต้องมี <conditions> ให้ทุก route fallback ไปที่ index.html (ตามที่คุยกันไว้ก่อนหน้า)
