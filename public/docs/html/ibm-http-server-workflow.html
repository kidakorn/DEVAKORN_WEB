💡 Key Point

  • HTTP Server รันอยู่บน IBM i เครื่องเดียวกับ AS/400 — ไม่ใช่ Windows Server แยก
  • IBM ใช้ Apache HTTP Server บน PASE (PASE = Unix-like environment บน IBM i)
  • ไฟล์ทั้งหมดอยู่ใน IFS (Integrated File System) ซึ่งก็คือ file system ของ IBM i

IFS Path Structure

ที่ Admin map กับ URL http://as400xxx/...
# IFS บน IBM i (ที่ Admin ตั้งค่าไว้) /www/ # root ของ web server kidakorn/ # folder หลัก (as400xxx/kidakorn) htdocs/ # document root index.html # → http://as400xxx/kidakorn/ dev/ # ← folder ที่ให้คุณสร้าง index.html # → http://as400xxx/kidakorn/dev/ css/ style.css js/ app.js logs/ # access log, error log conf/ # httpd.conf (Admin ดูแล) # URL mapping: http://as400xxx/kidakorn/ → /www/kidakorn/htdocs/ http://as400xxx/kidakorn/dev/ → /www/kidakorn/htdocs/dev/

⚠ Path จริงอาจต่างกัน

  • Path ด้านบนเป็นตัวอย่าง — path จริงขึ้นอยู่กับที่ Admin config ใน httpd.conf
  • ให้ถาม Admin ว่า "IFS path ที่ map กับ /kidakorn คืออะไร?"
  • หรือ login ACS → Tools → 5250 → พิมพ์ WRKLNK '/www/*'

สรุป

🎯 สิ่งที่รู้แน่ๆ ตอนนี้

01 — HTTP SERVER

IBM HTTP Server (Apache on PASE) รันอยู่บน IBM i เครื่องเดียวกับ AS/400 — ไม่ใช่ Windows Server แยก

02 — STATIC FILES

วาง HTML/CSS/JS ใน /www/kidakorn/htdocs/dev/ แล้วเปิดได้ที่ http://as400xxx/kidakorn/dev/ ทันที

03 — IWS POSSIBLE

เพราะ HTTP Server รันบน IBM i อยู่แล้ว — IWS เปิดได้ง่ายกว่าเดิมมาก ถ้า Admin ยอม = ไม่ต้องมี Backend เลย

04 — NEXT STEP

ถาม Admin เรื่อง IWS + IFS path จริง แล้วเริ่ม develop Frontend บน PC ก่อน ค่อย deploy ขึ้น IFS
