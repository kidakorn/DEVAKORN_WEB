Part 1 — Admin ทำอะไรบ้าง ตั้งแต่ต้น

🔴

สิ่งที่ Admin ต้องทำก่อน Developer เริ่มงาน

ทั้งหมดนี้ทำผ่าน ACS → IBM Web Administration for i
1

สร้าง HTTP Server Instance
ACS → IBM Web Administration for i → Create HTTP Server
กำหนดชื่อ instance เช่น KIDAKORN และ Port เช่น 8080 หรือ 80

2

สร้าง IFS Folder Structure
IBM i จะสร้าง folder ให้อัตโนมัติใน IFS:
/www/KIDAKORN/htdocs/ → Document Root
/www/KIDAKORN/logs/ → Access + Error log
/www/KIDAKORN/conf/ → httpd.conf

3

แก้ไข httpd.conf — กำหนด URL mapping
DocumentRoot "/www/KIDAKORN/htdocs"
Alias /kidakorn "/www/KIDAKORN/htdocs"
ทำให้ http://as400xxx/kidakorn/ ชี้ไปที่ IFS path นั้น

4

สร้าง Subfolder ให้ Developer
สร้าง /www/KIDAKORN/htdocs/dev/ ให้แต่ละ developer
ให้สิทธิ์ write เข้า folder นั้นได้

5

Start HTTP Server
STRTCPSVR SERVER(*HTTP) HTTPSVR(KIDAKORN)
หรือผ่าน ACS → Start HTTP Server

6

เปิด IWS (ถ้าต้องการ)
ACS → IBM Web Administration → Web Services → Enable
กำหนด URL prefix เช่น /web/services/

⚙️

httpd.conf — ไฟล์ config หลัก

Admin แก้ไขเพื่อ map URL → IFS path + เปิด IWS

⚠ Developer ไม่ต้องแตะ httpd.conf

  • Admin เป็นคนจัดการ httpd.conf ทั้งหมด
  • Developer แค่วางไฟล์ใน folder ที่ Admin กำหนดให้
  • ถ้าต้องการ path ใหม่หรือ config พิเศษ — ขอ Admin แก้ให้

✅ Developer ทำได้เอง

  • สร้าง/แก้ไขไฟล์ใน /htdocs/dev/
  • สร้าง subfolder เพิ่มได้ตามต้องการ
  • วาง HTML, CSS, JS, images ได้เลย
  • Build Angular/React แล้ว copy dist/ มาวาง

🚫 Developer ทำไม่ได้

  • แก้ httpd.conf (ต้อง restart HTTP Server)
  • สร้าง HTTP Instance ใหม่
  • เปิด/ปิด IWS
  • เข้า folder ของ dev คนอื่น

1

เปิด IBM Web Administration for i
ACS → Tools → IBM Web Administration for i
หรือ Browser: http://as400xxx:2001/HTTPAdmin

2

เลือก HTTP Instance
เลือก instance ที่สร้างไว้ เช่น KIDAKORN
ไปที่ tab "Web Services"

3

New Web Service
เลือก Program (ไม่ใช่ SQL)
ระบุ Library: X0250661 และ Program: PSS001R

4

กำหนด Parameters
P_REQNO → IN, CHAR(10)
P_RESULT → OUT, CHAR(1)
P_ERRMSG → OUT, VARCHAR(100)

5

กำหนด URL + HTTP Method
Service Name: issueReq
Method: POST
URL จะเป็น: /web/services/issueReq

6

Deploy
กด Deploy — ระบบจะ start service ให้อัตโนมัติ
ทดสอบที่: http://as400xxx/web/services/issueReq

ผลลัพธ์หลัง Deploy IWS
# URL ที่ได้หลัง deploy POST http://as400xxx/web/services/issueReq # Request body (JSON) { "P_REQNO": "REQ001" } # Response (JSON) { "P_RESULT": "S", "P_ERRMSG": "Issued successfully: REQ001" } # Services อื่นๆ ที่ทำได้เช่นกัน POST http://as400xxx/web/services/approveReq POST http://as400xxx/web/services/receivePO GET http://as400xxx/web/services/getStock

✅ ข้อดีของ IWS

  • ไม่ต้องมี Node.js/.NET backend เลย — ลด complexity มาก
  • RPG รันบน IBM i โดยตรง — เร็วและเสถียร
  • JSON in/out — Frontend เรียกได้ทันที
  • Admin จัดการ version และ security ผ่าน GUI

Part 3 — เรียก IWS จาก Frontend

📜

Vanilla JavaScript — fetch API

ใช้กับ HTML ธรรมดา ไม่ต้องใช้ Framework
// ────────────────────────────────────────────── // config.js — กำหนด Base URL ของ IWS // ────────────────────────────────────────────── const API_BASE = 'http://as400xxx/web/services'; // ────────────────────────────────────────────── // api.js — wrapper function สำหรับเรียก IWS // ────────────────────────────────────────────── async function callIWS(serviceName, body = {}) { try { const res = await fetch(`${API_BASE}/${serviceName}`, { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify(body) }); if (!res.ok) throw new Error(`HTTP ${res.status}`); return await res.json(); } catch (err) { console.error(`IWS Error [${serviceName}]:`, err); throw err; } } // ────────────────────────────────────────────── // ใช้งานจริง — Issue Requisition // ────────────────────────────────────────────── async function issueReq(reqNo) { const data = await callIWS('issueReq', { P_REQNO: reqNo }); if (data.P_RESULT === 'S') { alert(`✅ สำเร็จ: ${data.P_ERRMSG}`); } else { alert(`❌ Error: ${data.P_ERRMSG}`); } } // ────────────────────────────────────────────── // ดึง Stock List (GET) // ────────────────────────────────────────────── async function getStock() { const data = await fetch(`${API_BASE}/getStock`).then(r => r.json()); // data = [{ CDITEM, NMITEM, QTONHAND, ... }, ...] renderTable(data); } // ────────────────────────────────────────────── // Render ข้อมูลเป็นตาราง HTML // ────────────────────────────────────────────── function renderTable(items) { const tbody = document.querySelector('#stock-table tbody'); tbody.innerHTML = items.map(i => ` <tr> <td>${i.CDITEM}</td> <td>${i.NMITEM}</td> <td>${i.QTONHAND}</td> <td class="${i.QTONHAND <= i.QTMIN ? 'low' : ''}"> ${i.QTONHAND <= i.QTMIN ? '⚠ Low' : 'OK'} </td> </tr>`).join(''); }

Angular — HttpClient Service

สร้าง Service class เรียก IWS — แยก concern ชัดเจน
📄 iws.service.ts
import { Injectable } from '@angular/core'; import { HttpClient, HttpHeaders } from '@angular/common/http'; import { Observable } from 'rxjs'; import { catchError } from 'rxjs/operators'; // ── Interfaces ────────────────────────────────── export interface IwsResponse { P_RESULT: string; // 'S' = Success, 'E' = Error P_ERRMSG: string; } export interface StockItem { CDITEM: string; NMITEM: string; QTONHAND: number; QTMIN: number; TXUNIT: string; } // ── Service ───────────────────────────────────── @Injectable({ providedIn: 'root' }) export class IwsService { private readonly BASE = 'http://as400xxx/web/services'; private headers = new HttpHeaders({ 'Content-Type': 'application/json' }); constructor(private http: HttpClient) {} // Issue Requisition issueReq(reqNo: string): Observable<IwsResponse> { return this.http.post<IwsResponse>( `${this.BASE}/issueReq`, { P_REQNO: reqNo }, { headers: this.headers } ); } // Approve Requisition approveReq(reqNo: string, approvedBy: string): Observable<IwsResponse> { return this.http.post<IwsResponse>( `${this.BASE}/approveReq`, { P_REQNO: reqNo, P_APPROVED_BY: approvedBy }, { headers: this.headers } ); } // Get Stock List getStock(): Observable<StockItem[]> { return this.http.get<StockItem[]>(`${this.BASE}/getStock`); } }
📄 stock.component.ts — ใช้ Service
import { Component, OnInit } from '@angular/core'; import { IwsService, StockItem } from '../services/iws.service'; @Component({ selector: 'app-stock', templateUrl: './stock.component.html', }) export class StockComponent implements OnInit { items: StockItem[] = []; loading = false; error = ''; constructor(private iws: IwsService) {} ngOnInit() { this.loadStock(); } loadStock() { this.loading = true; this.iws.getStock().subscribe({ next: data => { this.items = data; this.loading = false; }, error: err => { this.error = err.message; this.loading = false; } }); } onIssue(reqNo: string) { this.iws.issueReq(reqNo).subscribe(res => { if (res.P_RESULT === 'S') { alert(`✅ ${res.P_ERRMSG}`); this.loadStock(); // refresh } else { alert(`❌ ${res.P_ERRMSG}`); } }); } isLow(item: StockItem) { return item.QTONHAND <= item.QTMIN; } }

💡 ถ้าเกิด CORS Error

  • Browser block request ข้าม origin — Admin ต้องเพิ่มใน httpd.conf:
  • Header always set Access-Control-Allow-Origin "*"
  • หรือระบุ origin เฉพาะ: Access-Control-Allow-Origin "http://as400xxx"
  • ถ้า Frontend อยู่บน IBM i เดียวกัน — ปกติไม่มี CORS ปัญหา

สรุป — ใครทำอะไร

งาน ใครทำ ทำที่ไหน รายละเอียด
สร้าง HTTP Instance 🔴 Admin ACS → IBM Web Admin ทำครั้งเดียว กำหนดชื่อ instance และ port
แก้ httpd.conf 🔴 Admin IFS /www/KIDAKORN/conf/ DocumentRoot, Alias, CORS headers
สร้าง IFS Folder ให้ Dev 🔴 Admin ACS → IFS Browser สร้าง /htdocs/dev/ และให้สิทธิ์ write
Deploy IWS Service 🔴 Admin ACS → IBM Web Admin → Web Services Expose RPG Program เป็น REST API
Start/Stop HTTP Server 🔴 Admin STRTCPSVR / ENDTCPSVR หลังแก้ config ต้อง restart
วางไฟล์ HTML/CSS/JS 🟢 Developer IFS /htdocs/dev/ ผ่าน ACS หรือ Network Drive ทำได้เองทันที ไม่ต้องขอ Admin
เรียก IWS API จาก JS 🟢 Developer Frontend code (fetch / HttpClient) ทำได้เองหลัง Admin deploy IWS แล้ว
เขียน RPG/CL Program 🔵 Developer (RPG) VS Code + Code for IBM i เขียนและ compile บน IBM i
